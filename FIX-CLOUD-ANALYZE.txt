FIX CLOUD ANALYZE (swimiqapp.com + Android)
===========================================

WHY IT FAILS
- Website + Android both call Supabase function: analyze-swim-video
- Live server is still OLD: 2026-gemini-sync-v9
- Google AI key/model is rejecting or overloading requests
- That is why you see busy / temporarily unavailable

THIS IS NOT FIXED BY:
- Paying GoDaddy SSL
- Cloudflare alone
- Uploading random zips without the server update


DO THESE 3 STEPS (in order)


STEP 1 — NEW GOOGLE AI KEY (2 minutes)
--------------------------------------
1. Phone or PC: https://aistudio.google.com/apikey
2. Sign in with the Google account you use for SwimIQ
3. Create API key
4. Choose "Create API key in a new project" (important)
5. Copy the key (starts with AIza...)


STEP 2 — PUT KEY IN SUPABASE (2 minutes)
----------------------------------------
1. Open https://supabase.com/dashboard
2. Open project: bryurwyeosbffvfpdpbv
   (or whatever shows as SwimIQ / StrokeIQ)
3. Left menu: Project Settings (gear) → Edge Functions
   OR: Edge Functions → Secrets / Manage secrets
4. Find secret name: GEMINI_API_KEY
5. Paste the NEW AIza... key → Save
6. Do NOT add GEMINI_MODEL secret (leave that deleted/absent)


STEP 3 — REDEPLOY THE VIDEO SERVER (5–15 minutes)
-------------------------------------------------
On the Windows PC (Desktop\StrokeIQ folder):

EASIEST if the bat exists:
  Double-click:  FIX-VIDEO-413-NOW.bat
  OR:            KARA-GEMINI-FIX-NOW.bat
  OR:            swimiq\KARA-GEMINI-FIX-NOW.bat

It will:
  - ask you to log into Supabase in a browser
  - deploy analyze-swim-video
  - should finish on version: 2026-gemini-sync-v12

If bats are missing (OneDrive deleted them), open PowerShell and paste:

  cd $env:USERPROFILE\Desktop\StrokeIQ\swimiq
  if (-not (Test-Path .\pubspec.yaml)) { cd $env:USERPROFILE\Desktop\StrokeIQ\StrokeIQ\swimiq }

  npx --yes supabase functions deploy analyze-swim-video --project-ref bryurwyeosbffvfpdpbv


STEP 4 — PROVE IT WORKED
